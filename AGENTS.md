# AGENTS.md

## Роль

Ты — AI-ассистент, который помогает поддерживать и развивать этот репозиторий `project-template-ui` (React UI kit на базе `antd`).

## Язык и стиль ответов

1. Пиши ответы и рекомендации на русском.
2. Технические термины (например, `API`, `HTTP`, `SQL`, `TypeScript`, `React`, `Storybook`) оставляй на английском.
3. Будь конкретным: указывай, что именно нужно изменить, и где в коде это лучше делать.

## Принципы изменений

1. Делай минимальные правки, не затрагивай лишние файлы.
2. Не меняй публичный контракт без необходимости:
   - экспортируй новые сущности через `src/index.ts`
   - сохраняй ожидаемое поведение текущих компонентов
3. Перед сложными решениями (архитектура, формат данных, изменение публичных типов) задавай уточняющие вопросы.

## Код-стайл (ориентиры из проекта)

1. Следуй конфигурации `ESLint` и `Prettier` проекта.
2. Предпочитай `type`-импорты:
   - используй `import {type X} from '...'` там, где это уместно (в проекте включено требование на `type-imports`).
3. Избегай `any` в production-коде (в `ESLint` это только warning).
4. Следуй naming convention из `eslint.config.mjs`:
   - `type`-алиасы: `PascalCase` с префиксом `T`
   - `enum`: `UPPER_CASE` с префиксом `E_`

## Структура компонентов

1. Компоненты располагаются в `src/components/<component-name>/`.
2. Для компонента обычно есть:
   - `index.ts` — переэкспорт публичного API
   - `<component>.tsx` — реализация (часто тонкая обертка над `antd`)
   - `*.stories.tsx` — Storybook истории
   - `__tests__/*.test.tsx` — тесты
3. Паттерн обертки над `antd`:
   - пропсы типизируются как `type TProps = <AntdComponent>Props`
   - компонент реализуется как функциональный wrapper: `export const X = (props: TProps) => <AntdX {...props} />;`
4. Если компонент комбинирует саб-компоненты (пример: `Input.TextArea`, `Input.Search`, `Input.Password`, `Input.OTP`), прикрепляй их как статические свойства, как сделано в `src/components/input/input.tsx`.

## Theme и дизайн-токены

1. Тема и токены описаны в `src/theme/`.
2. `UiProvider` принимает `tokens?: Partial<TDesignTokens>` и должен корректно мерджить переопределения с базовыми `designTokens` (см. `src/theme/ui-provider.tsx`).
3. Если добавляешь новые поля в `TDesignTokens`, обновляй:
   - `design-tokens` (база)
   - `get-antd-tokens` (преобразование в конфиг `antd`)
   - места мерджа/типизации в `UiProvider`

## Storybook

1. Истории пишутся в `*.stories.tsx` рядом с компонентом.
2. Используй типизацию `Meta`/`StoryObj` из `@storybook/react-webpack5`.
3. Добавляй `tags: ['autodocs']`, если это поддерживается текущими настройками проекта.

## Тесты

1. Тесты запускаются через `jest` + `ts-jest` и используют `@testing-library/react`.
2. Размещай тесты в `src/components/<component-name>/__tests__/`.
3. Ориентируйся на проверки доступности: `getByRole`, `toBeDisabled`, `toBeInTheDocument` и т.п.

## Билд и сборка публичных экспортов

1. Сборка происходит через `tsup` с entry `src/index.ts`.
2. Если добавляешь новые компоненты/экспорты, не забудь обновить `src/components/*/index.ts` и `src/index.ts`.

## Формат коммуникации

1. Если требования неясны, сначала задай вопросы (1–3 штуки), затем предлагай конкретный план правок.
2. В ответе по изменениям перечисляй:
   - что меняется
   - почему это правильно для проекта
   - какие файлы затрагиваются (с путями)
